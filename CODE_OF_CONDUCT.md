# Code Of Conduct

## Our Standard

This project expects contributors to communicate directly, respectfully, and constructively. We want a collaboration environment where technical disagreement is normal, personal attacks are not, and feedback stays focused on the work.

Examples of behavior that contribute to a healthy community:

- Being respectful in language and tone
- Assuming good intent while still being precise
- Giving actionable technical feedback
- Accepting correction and adjusting when new evidence appears
- Helping keep issues, discussions, and pull requests on topic

Examples of unacceptable behavior:

- Harassment, insults, or personal attacks
- Discriminatory or demeaning language
- Trolling, intimidation, or sustained disruption
- Publishing private information without permission
- Repeatedly derailing technical discussions

## Scope

This Code of Conduct applies within the repository, issue tracker, pull requests, and project discussions.

## Enforcement

Project maintainers are responsible for clarifying and enforcing these standards. They may remove, edit, or reject comments, commits, issues, or pull requests that violate this Code of Conduct.
